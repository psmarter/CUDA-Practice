# 深入 CUDA 优化：07. 万倍加速揭秘：FP16、INT8 与算力微操技术

在经历过之前对内存、共享内存，甚至是纯硬件寄存器通讯原语（Warp Shuffle）的疯狂压榨后，我们现在把视线投向了深度学习推理端最耀眼的另一领域——数据压缩与混合精度计算（Quantization）。
如果说之前的优化是在为 $FP32$ 的长宽高努力“铲土”，那么量化带来的则是直接让土的体积变水，密度激增。本篇博客我们将进入 `07_Quantization`，以我们在 RTX 4090 跑出的 `9.7 TFLOPS` (FP16) 与 `11.4 TOPS` (INT8) 这两组成绩，管窥 NVIDIA 对数值表示系统的天才级魔改内建指令（Intrinsic）。

## FP16 乘法的向量化魅力与 `half2`

在模型服务特别是 LLM 席卷之下，大模型的 weights 常以 `FP16` 或 `BF16` 铺开。这直接让内存占用减断到原来的一半，相当于带宽上限直接“无损”翻倍！
而在此之外更具有价值的是使用 NVIDIA 的 `half2` 指令集。当你以普通的方式拉取两个浮点数做相乘再相加时，你消耗的是一条纯粹的标量浮点管线指令（FMA）。但在 CUDA 中，如果我们可以把内存上相邻的两个 `half` 合并读作一个 32位的大结构体 `half2`，随后交由特殊的原生指令 `__hfma2` 来走——
你就直接完成了在**同一个时钟周期内，同时对两对 16 位浮点进行乘加**的魔法重构！（这就所谓的 SIMD 向量化并行）：

```cpp
// 一把拉起邻近的半精度数字包装位 32 位的 half2
half2 a_val = *((const half2*)&A[ ... ]);
half2 b_val = *((const half2*)&B[ ... ]);
// hfma2 原语，它等价于单独干了两次 f16 = f16 * f16 + f16
c_val = __hfma2(a_val, b_val, c_val);
```

正因为有了这样的向量解包级打包，我们的 FP16 向量化 GEMM 跑出了 `9723.10 GFLOPS` (近乎 10 TFLOPS) 的算力，相对于 CPU 版本加速超过极尽变态的五万倍！时间从上千毫秒被死死锚钉在了惊人的 `0.22 ms` 之内。

## INT8 与 `__dp4a` 内点积狂飙

如果 FP16 是加速界的敲门砖，INT8 则是模型落地的定海神针。由于矩阵算式本质无外乎是长向量内积的合算（Dot Product）。当你把原版 $32$-bit 的浮点按范围通过 `Scale` 参数映射进 $-127$ ~ $127$ 的紧凑 8 位整数格时，意味着你能用之前同样的存储带宽吞吐下 4 倍数量的矩阵！

这还不是全部。英伟达从 Pascal 架构起偷偷塞给开发者们一个绝对大杀器：`__dp4a` （Dot Product of 4 8-bit integers and Accumulate）。
它的可怕之处在于其硬软件的极致映射：它允许我们将 4 个连在一起的 8 位 `int8` 强行投胎为 1 个 32 位的 `int32` 拿在手上，进而触发特殊指令在一瞬间完成这四个元素的内项求和连乘：
$$ C_{\text{int32}} = C_{\text{int32}} + x[0]*y[0] + x[1]*y[1] + x[2]*y[2] + x[3]*y[3] $$ 
这一度让四次循环累加变魔术一样缩成为 1 次 ALU 内嵌运算。

从我们的测试平台 RTX 4090 输出成绩看，原生的 CPU 执行 $1024 \times 1024$ 规模耗时无穷无尽，而 INT8 `__dp4a` 向量化版本仅仅使用了 **`0.19 ms`** 便将其暴力清空！总体性能测定高歌猛进至了 **`11.40 TOPS`** (Tera Operations Per Second)。这就是脱离庞大开销后硬件底层整数乘法器的纯正战斗力。

## 更贴近业务：量化与反解机制 (Quantize & Dequantize)

有人会有疑虑：我们在计算的过程中使用的是 INT8 算完，但为了不溢出累加器往往是 INT32 的，最后我们需要交还给业务的往往又是 FP32 的对不齐的 Tensor，那他们之间的转换会不会变成新的“木桶短板效应”？

测试表明我们依靠 `Per-Tensor` 和 `Per-Channel` 的解包核函数配合强化的 `L2 Cache` 分配：
把浮点规模缩到 `INT8` (FP32 -> INT8)，单核耗时可以轻松实现 `2000~2900 GB/s` 级别的超大缓存击穿（速度甚至达到完全可忽略的几微秒），因为这种单纯标量运算直接命中了 4090 高达 72MB 的超大 L2 Cache，使得我们在模型推导的前后置阶段做格式强转基本处于零代价（Free Cost）。

## 结语

无论是大预言模型时代的 Flash Attention，还是 TensorRT 后端疯狂榨取的算力，无不是踩在这个 `half2 / int8 dp4a` 等硬件指令金字塔尖舞蹈。这一章的探秘展示了跳出传统 C 语言范式用物理方式（Bit reinterpret / union bytes packing）对待数据才能写出的奇迹。
接下来，我们将顺着这条道路，向着更高大上的结构进发。
